# Vercel이란?

## 한 줄 설명
> 웹 페이지를 인터넷에 올려주는 **무료 서비스**

## 핵심 특징
- GitHub에 코드를 올리면 → **자동으로 웹사이트 생성**
- 별도 서버 구매/관리 불필요
- 무료 플랜으로 충분
- 전 세계 어디서든 빠르게 접속 가능

## 비유
> GitHub = **문서 보관함**, Vercel = **자동 출판사**
> 보관함에 원고를 넣으면, 출판사가 알아서 책(웹사이트)을 만들어줍니다

# 배포 과정 — 딱 한 번만 설정

## 1단계: Vercel 가입 (1분)
- **vercel.com** 접속
- **Continue with GitHub** 클릭 → 로그인 끝

## 2단계: 프로젝트 연결 (2분)
- **Add New → Project**
- GitHub 저장소 선택
- **Root Directory** 에 `demo/bsc-dashboard` 입력
- **Deploy** 클릭

## 3단계: 완료!
- `https://내프로젝트.vercel.app` 주소가 생성됨
- 이 주소를 공유하면 누구나 접속 가능

# 이후 업데이트가 진짜 핵심

## 첫 배포 이후에는?

클로드 코드에게 이렇게만 말하면 됩니다:

```
"소스 반영해"
```
또는
```
"푸시해"
```

## 그러면 일어나는 일
1. 클로드가 변경사항을 GitHub에 push
2. Vercel이 자동 감지
3. 새 버전으로 **자동 재배포** (약 30초)
4. 웹사이트가 최신 상태로 업데이트

> 서버 접속, FTP 업로드, 빌드 명령어... **전부 필요 없습니다**

# 전체 구조 한눈에 보기

# 실습: Vercel 배포 체험

## 따라해보기 (10분)

1. 클로드에게: **"대시보드 커밋하고 푸시해줘"**
2. vercel.com 접속 → GitHub 로그인
3. Add New → Project → 저장소 선택
4. Root Directory: **demo/bsc-dashboard**
5. Deploy 클릭
6. 생성된 URL로 접속!

## 배포 후 업데이트 체험
1. 클로드에게: **"대시보드 제목을 '우리팀 공고 현황'으로 바꿔줘"**
2. 클로드에게: **"푸시해"**
3. 30초 후 웹사이트 새로고침 → 제목이 바뀌어 있음!

# 주의: Vercel 배포의 한계

## Vercel에 적합한 경우
- 오늘처럼 **조회 위주의 간단한 대시보드**
- 소개 페이지, 포트폴리오, 문서 사이트
- 데이터를 **보여주기만** 하는 경우

## Vercel로 안 되는 경우
- 로그인, 결제 등 **백엔드 로직**이 복잡한 서비스
- 파일 업로드, 실시간 채팅 등 **서버가 필요한 기능**
- 대용량 데이터 처리, 스케줄링 작업

## 정리
> 간단한 모듈이나 뷰어 → **Vercel로 충분**
> 규모가 커지면 → **별도 서버(AWS, NCP 등)가 필요**
>
> 오늘 배운 방식은 **빠른 프로토타입과 간단한 공유**에 최적입니다
