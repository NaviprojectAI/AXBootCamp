## 슬라이드 4 — HTML, CSS, JavaScript (p73)

![HTML CSS JS](slides/page_073.png)

### 스크립트

그래서 이 프론트엔드, 웹의 프론트엔드는 진짜 별거 없습니다. 이 세 가지 언어를 아시면 끝납니다. HTML, CSS, JavaScript. 이 세 개의 언어로 이 웹 개발이 구성이 되어 있습니다.

아니, 웹 개발 쉽다면서 무슨 언어를 세 개나 써가지고 개발하냐라고 생각이 들 수 있는데, 사실 크게 어렵지 않습니다. 심지어 문법은 이제 몰라도 됩니다. 역할만 알면 됩니다. 아, HTML이 이런 걸 하는 거고, CSS가 이런 걸 하는 거고, 자바스크립트가 이런 걸 하는 거다 정도만 알면 AI한테 "HTML 코드 만들어줘" 하면 만들어주고, "CSS 코드 만들어줘" 하면 만들어주고, 그 개념만 알면 되지 세부적인 문법은 이제 AI 시대에 알 필요가 없다라고 소개를 드립니다.

## 슬라이드 17 — JavaScript 이벤트 (p101~104)

![JavaScript 이벤트](slides/page_101.png)

## 참고: 우리 교육에서 달라지는 점

> 아래 내용은 발표 시 보충 설명용입니다.

### 1. claude.ai 링크 공유 vs Supabase + Vercel 연동

| 항목 | claude.ai 링크 공유 | Supabase + Vercel |
|------|-------------------|-------------------|
| 장점 | 즉시 공유 가능, 설치 불필요 | 본격적인 웹 서비스 구축 가능, DB 포함 |
| 단점 | 커스터마이징 불가, 도메인 없음 | 가입/연동 필요 |
| 적합한 용도 | 프로토타입 시연, 빠른 공유 | 실제 사용자가 쓰는 서비스 |

### 2. 외부 서비스 vs 직접 서버 구축

외부 서비스(Supabase, Vercel)를 이용하면 백엔드/프론트 서비스 구축이 편하지만, 실제 개발에서는 서버를 직접 구축해야 확장성이나 비용 면에서 무료로 사용이 가능합니다. 교육에서는 먼저 외부 서비스로 빠르게 경험하고, 이후 직접 구축하는 흐름으로 진행합니다.
